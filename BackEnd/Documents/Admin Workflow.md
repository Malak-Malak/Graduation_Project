### **Admin Workflow:**



###### **Admin logs in with seeded credentials (admin / Admin@123)**

###### **Admin can see all pending registration requests**

###### **Admin can see all approved/rejected requests history**

###### **Admin approves or rejects each request**

###### **If approved → system automatically creates a User account using the username and password from the university records table**

###### **If rejected → request status changes to Rejected, no account is created**

###### **Admin can see all current users on the system**

###### **Admin can add university records manually (for testing)**
